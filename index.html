Final Tutorial: Vinyl Release Popularity

Intro¶

Since the widespread adoption of streaming, the music industry has nearly left behind one of its most historic and special artifacts, the vinyl record. However, as the COVID-19 pandemic sweeps the world and people find both more time on their hands and a strong nostalgia for the way the world was, vinyl has found itself a large resurgence in the music marketplace. With this resurgence, people have once again found a quality in vinyl records that streamed music could never have, the joy of creating a collection.

With this tutorial I hope to start what is the long-term idea of bringing vinyl record collectors the tool of a vinyl record recommendation service. While there are nearly endless services to recommend the next album for you to steam or the next playlist for you to run to, there is no notable service to tell you whats the next vinyl to add to your collection. While this tutorial is not the full serivce, the exploration done here is the start of identifying good releases to recommend to collectors.

This tutorial uses a dataset retrieved from https://www.kaggle.com/ that contains a combination of data from https://www.discogs.com/ and Spotify. The Discogs website is geared towards vinyl collectors as a library of vinyl releases where each release is designated its own page, allowing users to view information and explore vinyl releases new and old. Since 2000, the site has accumulated nearly 2 million release submissions, cataloguing artwork, genre (albeit rigid), music style, label, and many more pieces of information per release. However, the most import aspect of Discogs to this tutorial is its want/have feature. Discogs allows users to mark releases as one they already have in their collection, or one they want to add soon. I will explain how I use this information more later, but this piece of information is most important to this tutorial as it speaks to how sought out a release is, as well as how rare the release may be. (https://www.kaggle.com/bethbaumann/first-edition-record-data-from-discogs)

Rarity and the amount a record is sought after are important aspects of a record to a collector. While some collectors may only care about finding records that match their music preferences, there are many who also seek impressive pieces to add to their collection. It is these people who would benefit most from a recommendation service of this caliber and that this tutorial is geared towards.

The Discogs Data¶

The dataset below will be the main focus of this tutorial. In this frame you will find many important pieces of information surrounding the top 1000 vinyl releases on Discogs. This information includes the artist, album, the number of albums the artist has made and the number of users who recorded having the release as well as the number of users who want the release. The frame also includes statistics such as the median price of the time since the release.

Within this frame I have also added an additional variable, Collector Value. This variable is how I will use this information in connection with the needs of a vinyl collector. The variable is simply the number of wants a release received divided by the number of people who reported having it. This variable captures the important traits I mentioned above that a collector looks for in a release by comparing the ratio of wants to haves. A higher scored release is one that had more wants than haves. This may indicate that a record is both rare and sought after, making it appealing to collectors.

This tutorial will focus mainly on analyzing this variable, and eventually attempting to predict it without knowing the haves and wants of a release.

Machine Learning¶

The idea behind the below machine learning program is to predict the Collector Value of a release without knowing the User Want and the User Have. To create this program, I used the top 5 most correlated features to Collector Value to train a k Nearest Neighbor regression. These variables are Median Price, User Rating, Number for Sale, Time Since Sold, and Years After First Album.

In [8]:
dfrLearn = dfr

#remove none values to allow the learning program to run properly
dfrLearn['median_price'] = dfrLearn['median_price'].fillna(dfrLearn['median_price'].mean())
dfrLearn['user_rating'] = dfrLearn['user_rating'].fillna(dfrLearn['user_rating'].mean())
dfrLearn['number_for_sale'] = dfrLearn['number_for_sale'].fillna(dfrLearn['number_for_sale'].mean())
dfrLearn['time_since_sold'] = dfrLearn['time_since_sold'].fillna(dfrLearn['time_since_sold'].mean())
dfrLearn['years_after_first_album'] = dfrLearn['years_after_first_album'].fillna(dfrLearn['years_after_first_album'].mean())

Before actually predicting any values, I made sure to find the most accurate k to use in my k-Nearest Neighbors regression. Doing so ensures that my predicted Collector Value will be as accurate as possible given my features and training data.

In [9]:
features = ['median_price', 'user_rating', 'number_for_sale', 'time_since_sold', 'years_after_first_album']
X_dict = dfrLearn[features].to_dict(orient="records")
y = dfrLearn['collector_value']


vec = DictVectorizer(sparse=False)
scaler = StandardScaler()

def get_cv_error(k):
    model = KNeighborsRegressor(n_neighbors=k)
    pipeline = Pipeline([("vectorizer", vec), ("scaler", scaler), ("fit", model)])
    mse = np.mean(-cross_val_score(
        pipeline, X_dict, y, 
        cv=10, scoring="neg_mean_squared_error"
    ))
    return k,mse
    
ks = pd.Series(range(1, 31))
ks.index = range(1, 31)
test_errs = ks.apply(get_cv_error)

kPrime = 0
temp = 1
for x in test_errs:
    if x[1] < temp:
        kPrime = x[0]
        temp = x[1]

With my optimal k found, I added space to enter the data of a release in question. While this program avoids using wants and haves, some of this data still needs to be collected from Discogs given that the values in the training data for these variables comes from Discogs. For example, the predicted Collector Value would be inaccurate if number of records for sale for the release in question came from Amazon, whos number for sale is likely much higher.

With the information entered, the Collector Value can be tested. As an example, I will predict the Collector Value of MTV Unplugged in New York by Nirvana.
