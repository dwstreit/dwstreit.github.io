Milestone 1

Writeup¶

For our final project, we will use data analysis to track music attributes, as well as estimate user appeal for randomly selected releases. We are considering using our own dataset collected from Discogs.com and the Last.fm Million Song Dataset. Discogs is the premiere website for music release data. Since 2000, the site has accumulated nearly 2 million release submissions, cataloguing artwork, genre (albeit rigid), music style, label, number of pressings, number collected, number wanted, and much more information. We will use user feedback as input to determine the likelihood that he or she enjoys an album from Discogs picked by random. We’re hoping to find how accurately we can use objective community submitted data to predict user's appreciation of a random work. https://www.discogs.com/search/?ev=em_rs&type=master To achieve this, we are developing an active web scraping algorithm to collect user-personalized datasets from Discogs.com to maximize efficiency, as well as to minimize run time and disk usage. Whereas we could attempt to archive the site data completely, it will take more time and effort than will be necessary to our end goal.

The Last.fm Million Song Dataset contains data including community-submitted genre tags, artist images, and number of listens and listeners. This data set is already collected and is stored in json files separated by three levels of single letter folders. Because we recently agreed to use this dataset, we are still developing code to utilize the information it provides. http://millionsongdataset.com/ A specific example, again using Nirvana, we can see that Smells Like Teen Spirit is the most popular song with over two million listens, and the most popular genre tag is Grunge. In addition, Nirvana has important artist information to those unfamiliar with the group. However, because Nirvana is a popular name among artists, there are multiple artists of the same name. For this issue, the genre tags may not be accurate, but the artist information lists multiple artists so as not to assume which Nirvana is selected. Artist information is listed numerically in no particular order. If our algorithm were to come across the fourth Nirvana in this list, the Dutch pop group formed in 1985, we will need to figure out how to accurately show only the information relating to the group and none relating to the popular Grunge group. https://www.last.fm/music/Nirvana

We also plan on exploring the Spotify API to analyze artist and song popularity, which are also important to a user who indicates they want mainstream recommendations.

Milestone 2

Writeup¶

Expanding on our findings from Milestone 1, we decided to incorporate the data 'dumps' provided by Discogs--archived xml files published each month under the data section of their website. https://data.discogs.com/?prefix=data/2020/ We used the most recent 'releases' xml dump, 'discogs_20200901_releases.xml.xml.gz'.

After downloading this bulk file, we used discogs-xml2db to extract and load the bulk release data into separated, tidy csv files. The data is seperated into csv format by general, artist, company, format, genre, identifier, image, label, style, track, track artist, and video data respectively. For our purposes, we're only using the general, genre, and style data. Although the genre and style data were ready to use, the general data needed adjusting.

Through this process we hope to create a way for vinyl record collectors to both explore music they might like, and also search for rare releases that they may want to pursue and add to their collections. Current music recommendation services fail to incorporate this second element, which is why we hope to build a platform which does. We're attempting to answer the question: can quantitative and categorical values of a release dictate similarities?
